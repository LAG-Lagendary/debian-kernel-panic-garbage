🎵 Итоговая инструкция по установке music21 в Linux (Debian/Ubuntu)

Эта инструкция помогает обойти ошибку externally-managed-environment, используя рекомендуемый метод — создание виртуального окружения Python (venv).

Шаг 1: Установка пакета python3-venv

В современных дистрибутивах Linux модуль для создания виртуальных окружений вынесен в отдельный пакет и отсутствует по умолчанию. Сначала нужно его установить.

Команда:

# Проверьте свою версию Python. Здесь используется 3.12 как пример.
sudo apt install python3.12-venv


Примечание: Если у вас другая версия Python (например, 3.10), замените python3.12-venv на соответствующий пакет (например, python3.10-venv).

Шаг 2: Создание и активация Виртуального Окружения

Виртуальное окружение (music_env) изолирует music21 от системного Python, устраняя ошибку externally-managed-environment.

2.1. Создание окружения

python3 -m venv music_env


Эта команда создаст папку music_env в вашем текущем каталоге, содержащую изолированную копию Python.

2.2. Активация окружения

Для работы с установленными в music_env библиотеками, окружение необходимо активировать.

source music_env/bin/activate


После успешной активации в начале вашей командной строки появится префикс (music_env).

Шаг 3: Установка music21

Теперь, когда окружение активно, вы можете безопасно установить библиотеку.

Команда:

pip install music21


Шаг 4: Деактивация окружения

После завершения работы с библиотекой, окружение можно деактивировать.

Команда:

deactivate


Префикс (music_env) исчезнет, и вы вернетесь в обычное системное окружение.

🚀 Использование Алиаса для быстрого запуска скрипта

Чтобы не активировать окружение вручную каждый раз, когда вам нужно запустить скрипт, использующий music21, вы можете создать алиас (псевдоним) в Bash.

Предположим, у вас есть скрипт my_script.py в вашей домашней папке (~).

4.1. Добавление алиаса

Откройте конфигурационный файл вашего шелла (например, .bashrc или .zshrc):

nano ~/.bashrc


(Или используйте vim, gedit, и т.д.)

Добавьте в конец файла следующую строку, которая напрямую вызывает интерпретатор из вашего виртуального окружения для запуска скрипта:

alias run_music='~/music_env/bin/python ~/my_script.py'


Сохраните файл и закройте его.

4.2. Активация алиаса

Чтобы изменения вступили в силу без перезагрузки терминала:

source ~/.bashrc


4.3. Запуск скрипта

Теперь, чтобы запустить ваш скрипт my_script.py с использованием music21, вам просто нужно ввести алиас:

run_music


Библиотека music21 будет доступна, так как скрипт выполняется напрямую через Python-интерпретатор из music_env.
