Настройка Скрипта AICreateMusic (ACM)

Это руководство поможет вам настроить рабочее окружение для запуска AI-модели Gemma 3 27B с помощью Python-скрипта acm_runner.py и удобного Bash-алиаса acm.

1. Установка Необходимых Компонентов

Для работы GGUF-моделей вам потребуется библиотека llama-cpp-python. Важно: Для использования ускорения GPU (NVIDIA/CUDA), вам может потребоваться специальная установка.

A. Установите Python-библиотеку

Установите llama-cpp-python:
Если у вас есть GPU NVIDIA (рекомендуется), используйте следующую команду для установки с поддержкой CUDA:

# Сначала установите CUDA Toolkit и сопутствующие библиотеки
pip install llama-cpp-python[cuda]


Если у вас нет GPU или вы хотите использовать только CPU:

pip install llama-cpp-python


Примечание: Если вы используете модель Gemma 3 27B, настоятельно рекомендуется использовать GPU.

B. Сохраните Python-скрипт

Сохраните код из блока acm_runner.py в файл с именем acm_runner.py.

Переместите этот файл в удобное и постоянное место, например, в /usr/local/bin или в вашу домашнюю директорию:

mkdir -p ~/scripts
mv acm_runner.py ~/scripts/acm_runner.py


2. Настройка Bash-алиаса acm

Алиас acm позволит вам запускать скрипт одной командой, передавая все необходимые аргументы.

A. Определите путь к скрипту

Предположим, что вы сохранили скрипт по пути: /home/user/scripts/acm_runner.py (замените /home/user на ваш реальный домашний каталог).

B. Добавьте алиас в ваш файл конфигурации

Откройте ваш конфигурационный файл Bash (обычно это ~/.bashrc, ~/.bash_profile или ~/.zshrc, если вы используете Zsh):

nano ~/.bashrc


Добавьте следующую строку в конец файла, обязательно указав правильный путь к вашему скрипту acm_runner.py:

# Алиас для запуска AI-композитора (AICreateMusic)
alias acm='python3 ~/scripts/acm_runner.py'


C. Примените изменения

Закройте и сохраните файл, затем примените изменения:

source ~/.bashrc


или

source ~/.zshrc


3. Использование команды acm

Теперь вы можете запускать модель с помощью одной простой команды, следуя синтаксису:

acm [НазваниеМодели] [ПутьКФайлуМодели] [ПутьКФайлуJSON] "[Ваша Инструкция/Запрос]"


Пример использования:

Предположим, ваша модель gemma-3-27b-it-Q4_K_M.gguf находится в /models/gemma/:

acm gemma-3-27b-it-Q4_K_M.gguf \
    /models/gemma/gemma-3-27b-it-Q4_K_M.gguf \
    ./epic_theme.json \
    "Создай эпическую симфоническую тему в ре-миноре с темпом 140 BPM, используя струнные, медные и хор."


Что происходит при запуске:

Прогресс: В консоли будет отображаться процент выполнения генерации ([Прогресс]: XX%), а также служебные сообщения об ошибках и статусе.

Вывод: Как только генерация завершится, модель сохранит файл epic_theme.json (или любой другой файл, который вы указали) с чистым JSON-кодом, соответствующим схеме, определенной в скрипте.
